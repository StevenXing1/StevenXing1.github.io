Project Overview

During my internship at Lenovo, I developed an AI-powered customer service platform designed to automatically answer customer questions about Lenovo devices. The project involved building an end-to-end solution combining large language model fine-tuning, intent recognition, and intelligent query routing to create a robust customer support system.

This was a demonstration project that showcased the technical feasibility and established the foundation for future large-scale deployment. The system was designed to handle approximately 80% of common device-related issues while maintaining natural conversation capabilities.

Development Journey

Phase 1: Initial RAG Approach with LangChain

Started by implementing a Retrieval-Augmented Generation (RAG) system using LangChain with an external knowledge base containing Lenovo device documentation and troubleshooting guides. However, this approach faced significant accuracy challenges in understanding nuanced customer queries and providing precise, device-specific answers.

Phase 2: Model Fine-tuning with LoRA & P-Tuning v2

Pivoted to fine-tuning approach using actual Lenovo customer service QA conversations. Implemented LoRA (Low-Rank Adaptation) and P-Tuning v2 techniques on ChatGLM 7B and Llama-2 7B models, creating domain-specific versions optimized for Lenovo device support scenarios.

Phase 3: Addressing Catastrophic Forgetting

Discovered severe catastrophic forgetting issue where fine-tuned models lost their general conversational abilities and couldn't handle greetings or daily communication. Attempted parameter tuning to find optimal trade-offs but couldn't achieve satisfactory balance between domain expertise and general capabilities.

Phase 4: Intent Recognition Solution

Developed a novel two-stage architecture with an upstream intent recognition model (Logistic Regression + Gradient Boosting) to classify user queries into categories: greetings, daily conversation, or professional device questions. This allowed routing simple queries to the base model while directing technical questions to the fine-tuned specialist model.

Phase 5: Presentation & Documentation

Presented the complete solution to Lenovo's technology group, demonstrating the architecture, technical decisions, and performance improvements. Created comprehensive documentation for future development teams to scale the demo into production-ready systems with larger models.

Technical Implementation

Phase 1: LangChain RAG System

The initial approach leveraged LangChain's retrieval capabilities to create a knowledge-augmented response system:

  • Knowledge Base Construction: Compiled extensive documentation including device manuals, troubleshooting guides, technical specifications, and FAQ databases into a searchable vector store.
  • Retrieval Pipeline: Implemented semantic search using BERT-based embeddings to retrieve relevant context for each customer query.
  • Generation: Fed retrieved context to a base language model (ChatGLM) to generate responses grounded in documentation.

Challenge Encountered

Despite having comprehensive documentation, the system struggled with accuracy. Issues included: imprecise retrieval leading to irrelevant context, difficulty handling multi-step troubleshooting, inability to understand device-specific terminology variations, and poor performance on complex diagnostic scenarios.

Phase 2: LLM Fine-tuning Strategy

Shifted to a fine-tuning approach using real customer service data to create domain-specialized models:

  • Data Collection: Obtained high-quality QA conversation logs from Lenovo's customer service department, covering common device issues, troubleshooting procedures, and support interactions.
  • LoRA Implementation: Applied Low-Rank Adaptation to ChatGLM 7B and Llama-2 7B, updating only ~0.1% of parameters while preserving model efficiency. This enabled rapid iteration and memory-efficient training.
  • P-Tuning v2: Implemented P-Tuning v2 for parameter-efficient fine-tuning, adding trainable prompt tokens at each layer while keeping the base model frozen.
  • Training Process: Fine-tuned on device-specific troubleshooting scenarios, product knowledge, and support best practices using Lenovo's proprietary customer service data.

Technical Insight

Both LoRA and P-Tuning v2 proved effective for domain adaptation with limited computational resources. The fine-tuned models showed excellent performance on device-specific queries, achieving high accuracy on technical troubleshooting scenarios that the RAG system couldn't handle.

Phase 3: The Catastrophic Forgetting Problem

Fine-tuned models exhibited severe performance degradation on general tasks:

Problem Description

After fine-tuning on customer service data, models lost their ability to handle basic conversational interactions. They couldn't respond appropriately to greetings ("Hello", "Good morning"), struggled with simple daily communication, and forced technical responses even for casual queries. This made the system feel robotic and unusable for natural customer interactions.

  • Root Cause Analysis: The fine-tuning dataset consisted almost entirely of technical QA pairs without sufficient representation of general conversational patterns, causing the model to overfit on technical scenarios.
  • Parameter Tuning Attempts: Experimented with various hyperparameters including learning rate reduction, training step adjustments, LoRA rank modifications, and dropout regularization to balance domain expertise with general capabilities.
  • Mixed Dataset Experiments: Tried mixing general conversation data with technical QA pairs, but couldn't find an optimal ratio that maintained both conversational fluency and technical accuracy.
  • Limitation Identified: Recognized that parameter tuning alone couldn't fully resolve the trade-off between specialization and generalization within a single 7B model given the available data and computational constraints.

Phase 4: Intent Recognition Solution

Developed an innovative two-stage architecture to overcome the catastrophic forgetting challenge:

Solution Architecture

Instead of forcing a single model to handle all scenarios, implemented an upstream intent classifier to intelligently route queries. This allowed using the base model for general conversation and the fine-tuned specialist for technical queries, achieving the best of both worlds.

  • Intent Categories: Designed a classification system with three primary categories:
    • Greetings & Social Pleasantries (Hello, Thank you, Goodbye)
    • Daily Conversation & Casual Chat (Weather, general questions)
    • Professional Device Questions (Technical support, troubleshooting)
  • Model Architecture: Built a hybrid classifier combining Logistic Regression and Gradient Boosting:
    • Logistic Regression for interpretable, fast initial classification
    • Gradient Boosting for handling complex, ambiguous queries
    • Ensemble approach for robust performance across diverse inputs
  • Feature Engineering: Extracted linguistic features including keyword presence, query length, question type, technical term density, and conversational markers to enable accurate intent detection.
  • Training Data: Curated a balanced dataset of labeled customer interactions spanning all three categories, ensuring the classifier could distinguish between general and technical queries.
  • Performance Optimization: Achieved top-1 F1 score improvement from 78% to 92% through iterative model refinement, feature selection, and threshold tuning.

Results & Impact

Technical Performance

  • Intent Recognition Accuracy: Achieved 92% top-1 F1 score, up from 78% baseline, enabling reliable query routing with minimal misclassification.
  • Coverage: System successfully handled approximately 80% of common Lenovo device issues including laptops, tablets, accessories, and peripherals.
  • Response Quality: Fine-tuned models provided accurate, context-aware technical responses while base model maintained natural conversational flow.
  • Latency: Intent classification added minimal overhead (<50ms), maintaining acceptable response times for customer interactions.

Key Insights

Catastrophic Forgetting Mitigation

When fine-tuning small models (7B parameters) causes catastrophic forgetting, architectural solutions like intent-based routing can be more effective than attempting to balance capabilities through data mixing or hyperparameter tuning alone. This modular approach maintains both specialization and generalization.

Practical AI Deployment

Real-world AI systems often benefit from combining modern and classical ML approaches. The lightweight intent classifier (LogReg + GB) added minimal latency while solving a critical architectural challenge, demonstrating that simpler models still have important roles in production systems.

Future Development Path

As this was a demonstration project, the documentation provided a roadmap for Lenovo's team to scale the system:

  • Larger Model Deployment: Recommendations for fine-tuning larger models (13B-70B parameters) that may better balance domain expertise with general capabilities.
  • Production Infrastructure: Guidelines for deploying the intent-based routing architecture at scale with load balancing and caching.
  • Continuous Learning: Proposed framework for incorporating new customer service interactions to continuously improve both the intent classifier and response models.
  • Multilingual Support: Expansion strategy for supporting multiple languages across Lenovo's global customer base.
  • Performance Monitoring: Metrics and dashboards for tracking system accuracy, customer satisfaction, and identifying areas for improvement.
