Project Overview

The Alzheimer's Edge Project delivers real-time EEG neuro-feedback for clinical research into early Alzheimer's markers. Participants receive feedback driven by their own gamma-band (30–55 Hz) inter-hemispheric coherence — a measure that is highly sensitive to muscle artifacts and line noise, and that must be computed fast enough for the feedback loop to feel instantaneous.

My work covered the full stack of that loop: re-engineering the legacy MATLAB signal-processing suite into production C++17, training deep-learning models to strip muscle artifacts out of the EEG before coherence is computed, and shipping a browser-based demo through WebAssembly so collaborators could run the pipeline without installing anything.

Technical Areas

Real-Time C++17 Signal Processing

Rewrote the MATLAB online-processing suite in C++17: channel selection, resampling, mean removal, and gamma-coherence computation over a sliding LSL window. The rewrite cut end-to-end latency from 270ms to 38ms, moving the feedback loop below the threshold where users perceive lag.

Deep-Learning Artifact Removal

Built a 1-D U-Net with squeeze-and-excitation attention and a multi-head self-attention bottleneck that predicts ICA-cleaned F3/F4 channels from 30-channel artifact-laden input — reaching F1 0.92 for artifact detection and +18dB SNR without the manual parameter tuning that ICA workflows demand.

WebAssembly Edge Deployment

Compiled the C++ pipeline to WebAssembly, streaming 256Hz multi-channel EEG at 60fps under a 5MB heap. Collaborators run the full pipeline in a browser with no installation, which made the system easy to demo to clinical partners.

Detailed Contributions

MATLAB → C++17 Re-engineering

The original neuro-feedback system ran in MATLAB, which made the real-time loop too slow and tied deployment to a MATLAB license. I ported the online signal-processing module to C++17:

  • Online Windowing: Pulled a sliding window of EEG from the Lab Streaming Layer (LSL) stream and processed it continuously rather than in offline batches.
  • Signal Pre-processing: Implemented channel selection, resampling (500 Hz → 250 Hz), and mean removal as streaming operations on a 30-channel input matrix.
  • Gamma-Coherence Computation: Reimplemented cross-spectral coherency estimation and gamma-band extraction, validating numerical parity against the MATLAB reference on identical test inputs.
  • Latency Result: End-to-end latency dropped from 270ms to 38ms (−86%), keeping the feedback loop responsive enough for live clinical sessions.

Artifact-Removal Model Architecture

Muscle artifacts contaminate exactly the frequency band the feedback depends on, so the model has to clean the signal without flattening genuine gamma activity:

  • 1-D U-Net (~2.25M params): Squeeze-and-excitation attention blocks with a multi-head self-attention bottleneck and an optional residual mode, predicting ICA-cleaned F3/F4 from 30 artifact-laden channels.
  • Composite Loss: Combined time-domain MSE, a multi-scale gamma spectral term (30–55 Hz), an F3–F4 cross-spectral coherency term, and a LogPowerLoss anti-shrinkage term.
  • Training Strategy: Mixed-precision (AMP) training with gradient checkpointing, cosine annealing with warmup, stochastic weight averaging, and gradient accumulation to an effective batch size of 192.
  • Augmentation: Mixup, polarity flips, additive noise, amplitude scaling, and channel masking, applied at the segment level to improve cross-subject generalization.

Key Insight — Regression to the Mean

Pure MSE training shrinks predicted variance, which showed up as a uniform ~1.5 dB under-prediction across the power spectrum — the model was hedging toward the average signal. Adding a LogPowerLoss term that penalizes the log-RMS mismatch per channel corrected the amplitude bias and pulled raw-signal RMSE down from 5.31 µV to 4.83 µV.

Data Quality Engineering

Much of the accuracy gain came from data handling rather than architecture — noisy training files were actively teaching the model the wrong thing:

  • 60 Hz Outlier Exclusion: Identified sessions where the notch filter alone could not remove line noise (~95 of 395 training files) and dropped them before segmenting, along with diagnostic tooling to detect and visualize those cases.
  • Segment Quality Gating: Filtered segments on amplitude, flat-line, and high-frequency-dominance criteria so degenerate windows never reached the model.
  • Subject-Level Splits: Partitioned train/validation/test by subject rather than by segment, preventing the leakage that would have inflated validation scores.
  • Memory-Efficient Storage: Float16 segment storage with clipping cut the in-RAM dataset from ~13GB to ~3.3GB, letting training run on 16GB machines without cuDNN host-allocation crashes.

Two-Channel Clinical Variant

Clinical settings often cannot mount a 30-electrode cap, so I built a reduced variant predicting F3/F4 from just Fp1 and Fp2. The information-poor regime required rebalancing rather than reusing the 30-channel recipe:

  • Smaller Model: Reduced to ~560k parameters — larger models found degenerate solutions when the input carried less signal.
  • Amplitude Correction: Raised the LogPower weight from 0.08 to 0.35 to counter the stronger amplitude shrinkage.
  • Disabled Cross-Spectral Loss: The F3–F4 coherency term injected characteristic 2 Hz oscillation artifacts as the model tried to satisfy covariance constraints it lacked the information to meet honestly.
  • Refocused Spectral Pressure: Moved the spectral loss from 30–55 Hz to 5–25 Hz, since 2-channel input carries no honest signal above ~25 Hz — applying gamma-band pressure there manufactures artifacts rather than recovering them.

Lesson Learned

A loss term that improves a model in one data regime can actively fabricate signal in another. When the input genuinely lacks the information a loss term demands, the model satisfies that term by inventing structure — so the honest fix is to remove the constraint, not to tune around it.

Results & Evaluation

Gamma-Band Coherence Accuracy

Evaluated on 1-second gamma coherence (30–55 Hz) across the primary cohort (139 sessions), comparing successive pipeline versions:

Metric V1 (30ch) V2 best V3 ensemble V3 best (LogPower)
MSE 0.0185 0.0174 0.0128 0.0114
RMSE 0.1359 0.1319 0.1133 0.1066
MAE 0.0918 0.0988 0.0742 0.0682
Pearson r 0.284 0.493 0.619 0.666

Correlation with ground-truth coherence more than doubled from V1 to V3 (Pearson r 0.284 → 0.666), with the largest single gain coming from the LogPowerLoss term and the 60 Hz outlier exclusion.

Cross-Cohort Generalization

The model was validated on a second, independent cohort (235 sessions) never seen during training:

Cohort Sessions Coherence RMSE Pearson r Raw Signal RMSE (F3 / F4)
Primary (in-distribution) 139 0.1066 0.666 4.83 / 4.89 µV
Secondary (held-out cohort) 235 0.131 0.567 4.60 / 5.78 µV

Performance degrades only modestly on the held-out cohort, indicating the model learned generalizable artifact structure rather than cohort-specific noise signatures.
